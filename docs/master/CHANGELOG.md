# dCorps Hub Master Reference – Changelog

> Source: `docs/master/DCHUB_MASTER.md`  
> Purpose: Track versioned changes to the Master Reference and closely related schemas/specs.

## 1. How to use this file

- Record only **meaningful, versioned changes** (not typos or formatting).
- Group entries by **released version tag** of the Master Reference.
- For each entry, capture: date, author, summary, affected sections/files, and any related PR/issue links.

Suggested entry format:

```markdown
## vX.Y.Z – YYYY-MM-DD

- **Change ID**: CHG-XXXX
  - **Author**: Name or handle
  - **Summary**: One short sentence
  - **Details**: Optional bullet list
  - **Master sections**: 3.2, 7.1B
  - **Related files**: docs/spec/SPEC-CORE.md
  - **Links**: PR #123, issue #456
```

## 2. Unreleased changes

Use this section to log changes that have been approved and merged to the Master Reference but are not yet part of a tagged release.

## 3. Release history

Add entries here once `DCHUB_MASTER.md` is versioned beyond v0.1, following the suggested format above.
