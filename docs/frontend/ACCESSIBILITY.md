# Accessibility and UX Guidelines

**Document type**: Frontend guideline  
**Doc ID**: FE-ACCESSIBILITY  
**Status**: Final v0.1  
**Source repo**: dcorps-docs (`docs/frontend/ACCESSIBILITY.md`)

> Scope: Accessibility and UX requirements for the website, explorer, and app.

---

## 1. Accessibility baseline

- Target WCAG 2.1 AA compliance.
- Keyboard navigation for all interactive elements.
- Visible focus states and skip-to-content links.
- Color contrast ratio >= 4.5:1 for body text.

---

## 2. UX requirements

- Clear labels for on-chain facts vs derived views.
- Explicit risk disclosures where applicable.
- Graceful error states and offline indicators.

---

## 3. Testing checklist

- Screen reader checks for core flows.
- Keyboard-only navigation across all pages.
- Contrast checks on primary and secondary palettes.
