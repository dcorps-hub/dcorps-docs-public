# Website Content Guidelines

**Document type**: Website guideline  
**Doc ID**: WEBSITE-CONTENT-GUIDE  
**Status**: Final v0.1  
**Source repo**: dcorps-docs (`docs/website/CONTENT_GUIDELINES.md`)

> Scope: Rules to keep the public website aligned with the documentation set.

---

## 1. Publish and update dates

- Every page must include a publish date.
- Material edits must update the "last updated" date.

---

## 2. Disclosures

- Roadmap pages must include a high-level disclaimer.
- Token-related pages must link to public risk disclosures.

---

## 3. Doc alignment

- All claims should link to authoritative docs or specs.
- If a claim cannot be linked, it should be removed or qualified.

---

## 4. Review process

- Major copy changes require a quick docs alignment review.
- Legal pages require counsel review (see `docs/legal/WEBSITE_LEGAL_DRAFTS.md`).
